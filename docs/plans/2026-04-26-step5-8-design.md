**Owner:** Moiz

## Scope

Stages 5–8 from the master build order, grouped into two execution plans. Stage 9 (Cloudflare Tunnel) is out of scope — app stays local.

- **Plan A (stages 5+6):** post-submission analysis pipeline + Style/Comparison/Pattern/Mistake modes
- **Plan B (stages 7+8):** stats page, spaced repetition, daily problem, streak, lightweight mock interview

Both plans build on the existing schema, agent streaming wrapper (`lib/agent/stream.ts`), full-solution safety filter, and `CoachPanel`.

---

## Plan A — Analysis Pipeline + Style Mode

### Goal

On every accepted submission, fan out parallel agent calls and persist them to `analyses`. Surface results at `/analysis/[attemptId]`. Add Style mode to the coach panel (manual, not auto-triggered).

### Schema additions

```sql
CREATE TABLE IF NOT EXISTS analyses (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  attempt_id INTEGER NOT NULL REFERENCES attempts(id),
  kind TEXT NOT NULL CHECK(kind IN ('quality','complexity','comparison','pattern','mistake')),
  content_md TEXT NOT NULL,
  status TEXT NOT NULL CHECK(status IN ('pending','done','error')),
  created_at INTEGER NOT NULL,
  UNIQUE(attempt_id, kind)
);

CREATE TABLE IF NOT EXISTS mistakes (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL REFERENCES users(id),
  problem_id INTEGER NOT NULL REFERENCES problems(id),
  attempt_id INTEGER NOT NULL REFERENCES attempts(id),
  category TEXT NOT NULL,
  note TEXT NOT NULL,
  created_at INTEGER NOT NULL
);

CREATE TABLE IF NOT EXISTS pattern_counters (
  user_id INTEGER NOT NULL REFERENCES users(id),
  pattern TEXT NOT NULL,
  solved_count INTEGER NOT NULL DEFAULT 0,
  PRIMARY KEY (user_id, pattern)
);
```

### Modules

- `lib/analysis/pipeline.ts` — orchestrator. Reads attempt + problem context, kicks off five `query()` calls via `Promise.allSettled`, runs each output through the existing safety filter, upserts rows. Idempotent: returns existing rows if all kinds present.
- `lib/analysis/prompts.ts` — focused system prompt per kind.
- `lib/analysis/repo.ts` — `getByAttempt(id)`, `upsertAnalysis(...)`, `recordMistake(...)`, `bumpPattern(user_id, pattern)`.

### Routes & pages

- `POST /api/analysis/[attemptId]` — triggers pipeline, returns immediately with current rows (status `pending|done|error`). Idempotent.
- `GET /api/analysis/[attemptId]` — current state of all rows.
- `app/analysis/[attemptId]/page.tsx` — server component with five sections; client island polls every 2s while any row is `pending`. Renders markdown.
- `/problem/[slug]` integration: when `runTests` returns all-pass, auto-POST and show toast/link "View analysis →".

### Style mode in CoachPanel

- New mode `style` in the existing mode selector. Reuses `lib/agent/stream.ts`, separate system prompt focused on PEP 8, naming, idiomatic Python.
- Manual trigger only (no debounced auto-fire) — keeps token spend predictable.

### Safety

- Every analysis output passes through the existing full-solution detector before persistence. On `blocked`, store `content_md = "[blocked: full-solution leak]"` with `status='error'`.

### Tests

- Unit: pipeline composition, idempotency, safety wrap, SM-2-style scoring not relevant here.
- Integration: route handlers against test DB.
- Playwright (gated): submit a known-correct fixture, navigate to `/analysis/[attemptId]`, assert all five section headers render.

## Plan B — Engagement + Interview

### Goal

Add stats page, spaced-repetition queue, daily problem, streak counter, and a lightweight timed mock-interview flow.

### Schema additions

```sql
CREATE TABLE IF NOT EXISTS review_queue (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL REFERENCES users(id),
  problem_id INTEGER NOT NULL REFERENCES problems(id),
  due_at INTEGER NOT NULL,
  ease REAL NOT NULL DEFAULT 2.5,
  interval_days INTEGER NOT NULL DEFAULT 1,
  UNIQUE(user_id, problem_id)
);

CREATE TABLE IF NOT EXISTS daily (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL REFERENCES users(id),
  date TEXT NOT NULL,           -- YYYY-MM-DD UTC
  problem_id INTEGER NOT NULL REFERENCES problems(id),
  completed INTEGER NOT NULL DEFAULT 0,
  UNIQUE(user_id, date)
);
```

### Stage 7 — SR + Daily + Streak

- `lib/sr/sm2.ts` — pure `nextReview(prevEase, prevIntervalDays, grade) => {ease, intervalDays, dueAt}`. Grade derived from attempt outcome:
  - `5` accepted on first run, no hints used
  - `4` accepted, multi-attempt or used hints
  - `2` failed but compiled
  - `0` runtime error / didn't submit
- On every accepted submission: upsert `review_queue` row using SM-2.
- `lib/daily/pick.ts` — deterministic `pickDaily(userId, dateISO)` seeded by `(userId, date)`. Prefers due-for-review, falls back to unsolved NeetCode 150.
- Streak: SQL-derived consecutive UTC-day count with ≥1 accepted attempt.

### Stage 7 — Pages

- `/stats`:
  - solved-by-topic and solved-by-difficulty bars
  - mistake-pattern table (from `mistakes` + analyses)
  - pattern-mastery (from `pattern_counters`)
  - due-for-review list with "Start review" link to `/problem/[slug]`
- Dashboard additions:
  - today's daily-problem card (start/continue/completed states)
  - current streak number
  - next 3 due reviews

### Stage 8 — Mock interview (lightweight)

- `/interview` landing: "Start 30-min session" → picks random unsolved → redirects to `/problem/[slug]?mode=interview&duration=30`.
- Problem-page interview-mode behaviors:
  - Header countdown timer; auto-finalizes attempt on expiry
  - "Run tests" hides pass/fail counts (only "Compiled" / "Runtime error")
  - CoachPanel locked to new `interview` mode — clarifying questions only, no hints, no style critique
  - On end (timer or user-submit): trigger Plan A analysis pipeline + a sixth `interview_debrief` analysis kind summarizing process (time-to-first-test, hint count, etc.)
- Schema: `analyses.kind` CHECK extended to include `'interview_debrief'` (migration handled in Plan B).

### Tests

- Unit: `sm2.ts` (table-driven), daily-pick determinism, streak math.
- Integration: enqueue review on accept; daily endpoint; interview-mode attempt persistence.
- Playwright (gated): stats page renders sections; interview mode shows timer + hides pass counts.
